## 2) Asks

### 2.1 — Fix `PATCH /api/v1/inventory/categories/{id}` (P0)

**Request body** (FE already sends):

```json
{ "name": "Antihypertensives" }
```

**Expected response** — same shape as `POST` create:

```json
{
  "success": true,
  "data": { "id": "<uuid>", "name": "Antihypertensives", "productCount": 12 }
}
```

**Validation:**
- 400 if `name` is missing / blank / > 80 chars
- 409 if another category in the same tenant already has that name (case-insensitive). The FE pre-checks but a concurrent rename could race — please return 409 not 500.
- 404 if id doesn't exist in tenant
- Otherwise 200 / updated row

### 2.2 — Fix `DELETE /api/v1/inventory/categories/{id}` (P0)

**Expected response:** `204 No Content` on success.

**Constraints:**
- 409 if any product references this category. FE will refuse to send the request when `productCount > 0`, but again — concurrent edits exist. Return 409 with a useful message so FE can toast it; don't 500.
- 404 if id doesn't exist in tenant
- 204 otherwise

We do **not** want a cascade delete or a "ghost" rename to "Uncategorised" — the user moves products manually first. If you want to offer a `?reassignTo={otherCategoryId}` query param later, it's additive and we'll wire it.

### 2.3 — Add `productCount` to `GET /inventory/categories` response (P1)

FE already declares the field as optional and renders `—` when missing. Once you ship it, the category page shows the count and the Delete button correctly enables/disables.

**Cheapest query path:** `SELECT c.id, c.name, COUNT(p.id) FROM categories c LEFT JOIN products p ON p.category_id = c.id WHERE c.tenant_id = :tenant GROUP BY c.id`. Index on `products.category_id` if it's not already there.

**Shape:**

```json
{
  "success": true,
  "data": [
    { "id": "uuid", "name": "Analgesics", "productCount": 12 },
    { "id": "uuid", "name": "Anti-Malaria", "productCount": 4 }
  ]
}
```

## 3) Why this matters (for prioritisation)

Seb&Bayor pharmacy is the first tenant going live with a public website (see `STUDIO_DEV_ONBOARDING_SEB_BAYOR.md`). The public catalog query is grouped by category — "Analgesics", "Anti-Malaria", "Antibiotics", "Vitamins & Supplements" etc. They cannot meaningfully populate their inventory until they can name/rename/delete categories. Today they can only POST new ones (immutable list = unacceptable for ops).

A pharmacist working through their existing stock will create a typo'd category once and need to fix it. We're not shipping the pharmacy website live until rename works.
